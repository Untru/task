Предназначен для ведения основных настроек конфигурации.

Настройки можно открыть соответствующей командой в разделе Сервис.

В константах устанавливается:

  • На вкладке "Общие"
    • Применять важность в обращении - учитывает возможность использовать признак "Важно" при работе с обращениями; 
  • На вкладке "Время удаления записей в обращении"
    •  Устанавливается время в минутах на удалении добавленной записи в обращениях по "Статусам", "Сообщениям", "Исполнителям"
      "Присоединенным файлам". Если время записи будет меньше контролируемого, то запись удалить не возможно.
      Примечание: если параметр установить "0", то контроль удаления не используется;
  • На вкладке "Загрузка обращений по E-mail"
    • Автозагрузка обращений по E-mail - устанавливает признак загружать или нет обращения по электроной почте;
    • Учетная запись автозагрузки обращений - через какую учетную запись будет загружаться обращение;

      Примечание: Загрузка происходит через регламентное задание выполняемое каждые 900 секунд. Поэтому при файловом
      режиме работы БД: у одного из пользователей с полными правами (для выполнения регламентного задания) необходимо
      установить в "Дополнительные параметры запуска" значение "AllowExecuteScheduledJobs". Данная опция доступна на платформе 8.3.5.
